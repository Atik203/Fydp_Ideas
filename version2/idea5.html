Scientific Verification
via Agentic LLMs

A multi-agent LLM pipeline that autonomously decomposes, retrieves, audits, and verifies scientific claims — with a confidence-gated hallucination guard that refuses to issue verdicts without sufficient evidence.

Domain LLM + Agentic AI
Primary Venue ACL / NeurIPS 2026
Risk Level ⚠️ Scope Carefully
Compute Medium–High
Score 24 / 30
🤖 Why Agentic AI is Architecturally Central Here (Not Cosmetic): Scientific verification is an inherently multi-step, tool-dependent task. A single LLM inference cannot (1) decompose a paper into verifiable atomic claims, (2) retrieve live evidence from PubMed, (3) audit statistical methodology, and (4) gate confidence before issuing a verdict — simultaneously. Each step requires independent reasoning, external tool access, and conditional control flow. This is a textbook use case for agentic orchestration, not a decorated chat interface.
🔴 Real-World Problem Statement
  • 2.5 million scientific papers are published annually — manual claim-level verification is structurally impossible at this scale
  • LLMs confidently hallucinate scientific verdicts — particularly dangerous in medical, pharmaceutical, and climate policy contexts
  • No AI system can autonomously cross-check the experimental claims inside a paper against the broader literature
  • Peer review takes 6–18 months per paper; fraudulent or methodologically flawed papers circulate for years before retraction
  • Existing automated tools (Scite.ai, Semantic Scholar) perform citation context analysis — not autonomous claim-level verification with methodology auditing
📚 Existing Solutions & Their Limitations
  • AI Scientist (Lu et al., 2024): Generates scientific papers autonomously — does not verify existing claims; orthogonal task entirely
  • GPT-4 as reviewer: Single-pass inference; no external tool use; hallucination rate ~23% on FactScore; no confidence gating
  • PaperQA2 (Skarlinski et al., 2024): QA over retrieved papers — no claim decomposition, no methodology auditing, no multi-agent coordination
  • Scite.ai: Citation context classification (supporting/contrasting) — not autonomous verification; no structured verdict output
  • MultiVerS (Wadden et al., 2022): Multi-document NLI for scientific claims — no agentic tool use, no methodology reasoning
  • SciFact (Wadden et al., 2020): Single-sentence claim verification against abstracts — no full-paper decomposition
🔬 Explicit Research Gap
  • No agentic system exists that decomposes a paper into atomic verifiable claims as a first step of an autonomous pipeline
  • No framework combines retrieval + methodology auditing + confidence-gated hallucination control in a single multi-agent architecture
  • Hallucination control in multi-step, tool-using scientific reasoning is entirely unstudied — existing hallucination work targets single-inference settings
  • No structured verification report format exists as a community standard for automated scientific review outputs
  • Statistical methodology auditing (sample size adequacy, p-value validity, effect size reporting) via LLM agents has zero prior literature
✨ Proposed Solution — Key Components
  • Claim Decomposition Agent: Parses biomedical abstracts into atomic, independently verifiable claim units using structured prompting + GROBID
  • Evidence Retrieval Agent: Issues structured queries to PubMed + Semantic Scholar APIs; returns supporting and contradicting papers with relevance scores
  • Methodology Auditor Agent: Evaluates statistical validity — study design type, sample size adequacy, p-value reporting, effect size, confidence intervals
  • Hallucination Guard: Confidence-gated module — if evidence coverage < threshold, issues "Insufficient Evidence" rather than a fabricated verdict
  • Verification Report Generator: Produces structured, human-readable verification summary with per-claim verdicts, evidence chains, and confidence scores

Detailed Methodology
Datasets, models, training procedures, and experimental design

Datasets

  • SciFact (Wadden et al., 2020): 1,409 expert-annotated scientific claims — primary evaluation benchmark for claim verification accuracy
  • SciEval (Sun et al., 2023): Multi-task scientific reasoning — evaluates evidence retrieval precision and reasoning chain quality
  • PeerRead (Kang et al., 2018): 14,000+ peer review decisions — used for methodology flaw detection F1 evaluation
  • ExpertQA (Malaviya et al., 2023): Expert-curated Q&A with citation requirements — hallucination evaluation ground truth
  • FactScore (Min et al., 2023): Atomic fact-level hallucination scoring framework — primary hallucination evaluation protocol
  • Custom SciVerify-Bio: 500 biomedical abstracts with manually annotated claim decompositions — constructed in Semester 1

LLM Backbone Selection

  • Llama-3.1-70B (Meta, 2024): Primary backbone — open weights, reproducible, strong scientific reasoning
  • Mistral-Large-2 (Mistral AI, 2024): Comparison baseline — strong tool-use performance in agentic settings
  • GPT-4o (OpenAI): Upper-bound oracle baseline — not used in production due to reproducibility constraints
  • All models accessed via vLLM inference server for consistent latency and cost control

Agent Training & Prompting Strategy

  • Claim Decomposition Agent: Few-shot prompting with 15 manually annotated decomposition examples; evaluated by inter-annotator agreement (Cohen's κ)
  • Evidence Retrieval Agent: Structured query generation via chain-of-thought prompting; retrieval evaluated by NDCG@10 against SciFact gold evidence
  • Methodology Auditor Agent: Rubric-grounded prompting with explicit statistical validity checklist; validated against PeerRead expert decisions
  • Hallucination Guard threshold τ: Calibrated on held-out validation set using Platt scaling; threshold set to minimize FPR at fixed TPR=0.90

Ablation Study Design

  • Ablation A1: Remove Hallucination Guard → measure hallucination rate increase
  • Ablation A2: Remove Methodology Auditor → measure verdict accuracy degradation
  • Ablation A3: Single-agent vs. multi-agent → isolate coordination benefit
  • Ablation A4: GPT-4o backbone vs. Llama-3.1-70B → measure open-source viability
  • Ablation A5: 2-agent vs. 3-agent pipeline → determine optimal agent count

Novel Contributions — What Makes This New
Four independently publishable artifacts from one FYDP project
01
Confidence-Gated Hallucination Guard
First hallucination control mechanism designed specifically for multi-step agentic scientific reasoning. Unlike post-hoc hallucination detection (Min et al., 2023), this is an inline, proactive gate that prevents verdict generation when evidence coverage is insufficient. Formalized as a threshold function over evidence density and source authority scores.
AI Safety contribution → NeurIPS / ICLR
02
Methodology Auditor Agent
No prior work has used an LLM agent to evaluate statistical methodology validity (study design classification, sample size adequacy, p-value threshold compliance, effect size reporting completeness) as a step in a verification pipeline. This introduces a rubric-grounded, tool-augmented agent that acts as a proxy statistical reviewer — validated against PeerRead expert decisions.
Novel agent type → ACL / EMNLP
03
SciVerify-Bio Benchmark Dataset
500 biomedical abstracts with expert-annotated atomic claim decompositions, per-claim verification verdicts, evidence mappings, and methodology quality labels. No such structured benchmark exists for claim-level scientific verification. Released publicly under CC BY 4.0 — designed to become the standard evaluation resource for agentic scientific verification research.
Dataset paper → ACL / EMNLP / NeurIPS Track
04
Structured Verification Report Format (SVRF)
A standardized JSON + natural language report format for automated scientific verification outputs — per-claim verdict, evidence chain, methodology flag, confidence score, and "Insufficient Evidence" fallback. Proposed as a community standard analogous to the FEVER shared task output format — enables reproducible comparison across future systems.
Shared task proposal → ACL Anthology

Agentic AI Justification — Why Multi-Agent is Necessary, Not Decorative

Why a Single LLM Cannot Do This

  • A single inference cannot decompose AND retrieve AND audit AND gate AND synthesize within one context window — the tasks are sequentially dependent with external tool calls between steps
  • Real-time PubMed API calls require external tool integration — impossible in a single-pass inference setting
  • The Hallucination Guard must operate between reasoning steps — not as a post-hoc filter — requiring inter-agent communication
  • Methodology auditing requires a separate agent context to apply the statistical checklist rubric without contaminating the retrieval context

What Agentic AI Enables That Nothing Else Can

  • Tool use with conditional control flow: Agents query external APIs, receive results, and decide whether to proceed or escalate to the Hallucination Guard — requiring genuine multi-step planning
  • Specialized agent contexts: Each agent maintains its own focused context (decomposition / retrieval / auditing) — avoids catastrophic context mixing that degrades single-LLM performance
  • Iterative verification: If Claim_1 fails the Hallucination Guard, only Claim_1 is flagged — other claims proceed independently — true parallel agentic reasoning
  • Transparent reasoning traces: LangGraph captures the full multi-step reasoning graph — enabling interpretability analysis as a secondary contribution
