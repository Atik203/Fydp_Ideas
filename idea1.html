💡 1. Core Idea

We propose a multi-agent LLM debate framework where each participating agent is assigned a dynamic trust score updated in real-time based on the empirical verifiability of its claims against a retrieved scientific evidence corpus. Unlike standard Multi-Agent Debate (MAD) — where agents argue freely and converge via majority voting — our system introduces a Trust-Calibrated Adjudication layer: agents whose claims are contradicted by RAG-retrieved evidence receive mathematically reduced influence in subsequent rounds; well-grounded agents gain proportionally higher weight.

🎯 Core Research Contribution

The primary contribution is a mathematical formulation of per-agent trust within a multi-agent debate session — a concept absent from all existing frameworks. Secondary contributions include: (1) a RAG-grounded claim verification sub-module, and (2) a reproducible sycophancy measurement protocol applicable to any multi-agent system.

🔍 2. Problem Statement

2.1 The Real-World Problem

Multi-agent debate has emerged as a leading paradigm for improving LLM reasoning reliability (Du et al., 2023; Liang et al., 2023). The intuition is sound: multiple independent models challenge each other's reasoning, converging on correct answers through dialectical pressure. However, the paradigm suffers from a well-documented and dangerous failure mode — agentic sycophancy.

Sycophancy manifests when agents abandon correct initial positions to conform with a confidently-stated but incorrect majority opinion. Perez et al. (2023) demonstrated sycophancy rates of 40–55% on complex reasoning tasks when a planted incorrect majority opinion was introduced. Wei et al. (2024) further showed this behaviour persists even in the largest frontier models (including next-generation architectures like Gemini 3.1 Pro and GPT-5.5).

2.2 Engineering Complexity

From a systems perspective, this requires solving three interdependent challenges:

Challenge A: Real-time Verification

Decompose each agent's claim into atomic propositions and retrieve supporting/contradicting evidence within the debate turn latency budget (<10s per turn).

Challenge B: Trust Stability

Design a trust update rule that avoids degenerate equilibria — all trust collapsing to zero, or trust becoming dominated by the first correct agent regardless of subsequent performance.

Challenge C: Behavioural Effectiveness

Prompt-engineering trust weights such that they actually change model outputs, not just appear in the prompt context and get ignored by the model's attention mechanism.

Challenge D: Heterogeneous Agents

Extend the framework to agents from different model families (Llama-3, Mistral, Phi-3) — trust calibration must be model-agnostic.

📚 3. Existing Solutions & Limitations

⚠ Explicit Research Gap

No existing framework provides a mathematically formalized, evidence-grounded trust mechanism that dynamically re-weights agent influence based on the verifiability of their claims against an external corpus. The gap is not "multi-agent debate doesn't work" — it is that existing systems lack the algorithmic infrastructure to penalize hallucinating agents and reward evidence-aligned agents within the same debate session. This is a missing piece that, once formalised, generalises across any multi-agent reasoning system.

🏗️ 4. Proposed Architecture

4.1 Debate Agent Pool

N agents (N = 3 by default) instantiated from open-weight models. Each agent maintains: (a) its current position on the debate topic, and (b) a trust vector T ∈ ℝN representing its assigned credibility weight. At initialisation, all trust values are uniform: Ti,j = 1/N.

4.2 RAG Evidence Retrieval Module

After each debate turn, every claim made by the active agent is decomposed into atomic propositions (using an LLM prompt with structured JSON output) and queried against a domain corpus (Semantic Scholar API → PubMed/ArXiv abstracts). A cross-encoder reranker returns top-K evidence with a relevance score per claim. Claims with no supporting evidence above a threshold θ are flagged.

4.3 Trust Calibration Function (Core Contribution)

For agent i at debate round t, the trust update is:

Ti(t+1) = Ti(t) · σ( α · Vi(t) − β · Hi(t) )

Variables

  • Vi — Verification score: proportion of claims supported by evidence
  • Hi — Hallucination penalty: proportion of claims contradicted
  • α, β — Tunable hyperparameters (sweep range: 0.5–2.0)
  • σ — Sigmoid function, bounds trust in (0, 1)

Design Properties

  • Trust never collapses to zero (sigmoid lower bound > 0)
  • Trust floor Tmin = 0.1 prevents agent silencing
  • Trust ceiling Tmax = 0.9 prevents single-agent dominance
  • Update is session-scoped — resets between questions

4.4 Trust-Weighted Adjudication

Final answer is determined by trust-weighted aggregation over agent final positions, not majority vote. This is conceptually analogous to weighted ensemble methods but applied at the reasoning trace level — each agent's complete argument chain is weighted by its accumulated trust score Ti(K) at round K.

5.3 Experimental Protocol

  1. Baseline replication: Reproduce sycophancy rates from Du et al. (2023) using standard MAD (N=3, K=3) on SciBench. Inject planted incorrect majority opinions to measure sycophancy rate.
  2. Trust-calibrated system: Same setup with trust function active. Compare to all baselines.
  3. Full ablation matrix: (a) Trust only, no RAG grounding. (b) RAG grounding, no trust function. (c) Full system. (d) Standard MAD. (e) Single-agent CoT. (f) GPT-4o oracle.
  4. Hyperparameter sweep: α, β ∈ {0.5, 1.0, 2.0}; K ∈ {2, 3, 5} debate rounds.
  5. Heterogeneous agent experiment: Mix Llama-3 + Mistral + Phi-3 as debaters.

Statistical rigour: All experiments run across 3 random seeds. Results with 95% CI. Paired bootstrap significance testing (Koehn, 2004). Effect sizes (Cohen's d) reported alongside p-values.

⚠ Risks & Mitigations
  • Compute cost: Multi-agent inference is N× more expensive. Mitigation: 4-bit quantized 8B models via vLLM; cap N=3, K=3. Cost per question ≈ 9 forward passes — tractable on a single A100 for 500-question evaluation.
  • RAG quality bottleneck: Noisy evidence degrades trust signal. Mitigation: Cross-encoder reranker; citation-count filtering (>10 citations); restrict to domain-specific abstracts.
  • Trust instability: Sigmoid-based update may converge trivially. Mitigation: Trust floor Tmin=0.1; stability analysis in ablations; compare against simpler heuristic (running accuracy) as sanity check.
  • Novelty concerns at review: Sycophancy papers are increasingly common. Mitigation: Frame as the first system with a formal trust calibration function; emphasise mathematical formulation and trust ECE metric as novel diagnostics.
